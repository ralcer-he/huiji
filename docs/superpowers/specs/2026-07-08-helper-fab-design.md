# 助手悬浮窗重构设计

## 概述

重构现有的"小慧"悬浮窗功能，将其改为单一"助手"入口，支持智能识别用户意图创建记录、回答软件使用问题。根据AI等级自动切换识别模式。

## 目标

1. 简化界面：移除模式切换，改为单一助手入口
2. 智能识别：根据AI等级自动选择语义理解或关键词匹配
3. 创建记录：支持通过自然语言创建备忘、随笔、日记、心情记录
4. 软件帮助：回答软件使用问题
5. 用户反馈：提供友好的操作反馈和引导

## 界面设计

### 悬浮按钮
- 保持可拖拽功能
- 图标改为"assistant"或"sparkle"
- 显示AI可用状态

### 聊天窗口
- 标题改为"助手"
- 移除模式切换栏（助手/写信/树洞）
- 保留聊天对话界面
- 移除历史对话功能

## 意图识别逻辑

### 高级模式（AI可用时）
- 调用AI进行语义理解
- 识别意图类型：创建记录 / 软件帮助 / 闲聊
- 提取记录类型、内容、标题等信息
- 不确定时向用户确认

### 低级模式（AI不可用时）
- 关键词匹配规则：

| 记录类型 | 触发关键词 |
|---------|-----------|
| 备忘 | 备忘、待办、任务、提醒、需要做 |
| 随笔 | 随笔、记录、写点、想法（排除其他类型关键词） |
| 日记 | 日记、今天、今日、记录一天 |
| 心情 | 心情、情绪、感觉、累、烦、开心、难过、生气、焦虑 |
| 软件帮助 | 怎么、如何、哪里、帮助、使用、教程 |

### 心情记录模式判断
- **快速模式**（默认）：只记录情绪名称，强度默认3
- **详细模式**：用户提到"备注"、"详细"、"记录一下"、"说说"时，同时记录情绪和备注内容

### 标题判断
- 默认无标题（自动截取前50字符）
- 用户明确提到"标题"时设置标题

## 创建记录规则

### 备忘
- 提取内容，默认无标题
- 自动标记为未完成状态

### 随笔
- 提取内容，默认无标题

### 日记
- 提取内容，默认无标题
- 使用当前日期

### 心情
- 识别情绪词，匹配最接近的情绪
- 默认快速模式，强度为3
- 详细模式时记录备注内容

## 软件帮助FAQ（预设）

| 用户问题 | 助手回答 |
|---------|---------|
| 在哪里看历史记录？ | 去回忆页面查看所有历史记录 |
| 如何添加标题？ | 在编写页面顶部输入框添加标题（可选） |
| 怎么删除记录？ | 在回忆页面打开记录详情，点击删除按钮 |
| 如何分享记录？ | 在记录详情中点击分享按钮生成卡片 |
| 怎么切换主题？ | 在设置页面可以切换深色/浅色主题 |
| 如何添加标签？ | 在编写页面底部输入标签，用逗号分隔 |
| 心情记录在哪里？ | 在回忆页面或日历页面可以查看心情记录 |

## 反馈机制

### 成功反馈
- 创建备忘："备忘已添加！还有什么需要帮助的？"
- 创建随笔："随笔已记录！还需要我做什么吗？"
- 创建日记："日记已保存！今天过得怎么样？"
- 创建心情："心情已记录！照顾好自己哦~"
- 回答帮助：提供答案后，询问是否需要更多帮助

### 失败反馈
- 创建失败："抱歉，创建失败，请重试"
- 无法识别："我不太明白，你可以试试说'帮我写一个背单词的备忘'"

## 技术实现

### 组件结构
- `XiaohuiFab.jsx`：悬浮窗主组件（重命名为`HelperFab.jsx`或修改现有组件）
- 内部包含：
  - `FloatingButton`：悬浮按钮（保持不变）
  - `HelperChat`：聊天窗口（重构）

### 核心逻辑
1. 输入处理：接收用户输入
2. 意图识别：根据AI状态选择识别方式
3. 操作执行：创建记录或回答问题
4. 结果反馈：显示操作结果

### AI调用提示词（高级模式）
```
你是一个笔记应用的智能助手，负责帮用户创建记录和回答使用问题。

用户输入可能包含以下意图：
1. 创建备忘：包含"备忘"、"待办"等词，提取内容
2. 创建随笔：包含"随笔"、"记录"等词，提取内容
3. 创建日记：包含"日记"、"今天"等词，提取内容
4. 创建心情：包含情绪词，识别情绪类型和强度

请返回JSON格式：
{
  "intent": "create_note|create_memo|create_diary|create_mood|help|chat",
  "recordType": "note|memo|diary|mood",
  "content": "提取的内容",
  "title": "标题（可选）",
  "emotion": "情绪名称（心情类型）",
  "isDetailed": true/false,
  "confidence": 0-1
}

如果不确定，confidence设为0.5以下，并在content中说明疑问。
```

### 关键词匹配（低级模式）
- 使用正则表达式匹配关键词
- 根据匹配结果构建操作指令
- 不确定时返回引导信息

## 数据模型

### 记录创建接口
使用现有的`saveRecord`函数，传入以下结构：

```javascript
{
  id: string,
  type: 'note' | 'memo' | 'diary' | 'mood',
  content: string,
  title: string (可选),
  emotions: string[] (心情类型),
  intensity: number (心情强度),
  tags: string[],
  completed: boolean (备忘),
}
```

## 状态管理

### 组件状态
- `messages`：聊天消息列表
- `input`：用户输入
- `loading`：加载状态
- `aiStatus`：AI状态（等级、可用次数）

### 输入处理流程
```
用户输入
    ↓
判断AI是否可用
    ↓
[可用] 调用AI识别意图
[不可用] 关键词匹配
    ↓
识别意图类型
    ↓
[创建记录] 解析内容，调用saveRecord
[软件帮助] 查找FAQ或调用AI
[闲聊] 调用AI回复
    ↓
显示反馈消息
```

## 错误处理

1. **AI调用失败**：降级为关键词匹配模式
2. **网络错误**：提示用户检查网络，使用预设回答
3. **创建记录失败**：显示错误提示，建议手动创建
4. **意图无法识别**：显示引导示例
